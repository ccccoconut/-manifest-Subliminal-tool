# 酥饼 — 路演 / Demo 脚本（3 分钟）

> 面向 **手机 WebView / 移动端浏览器** 现场演示。功能以当前代码为准。

## 0. 一句话定位

> 我们不是让 AI 替你写一首歌，而是帮你把一句「我不行」，变成一段**用你自己声音读出**的「我可以」——你的第一条 **sub**。

## 1. WOW 在前 30 秒

**Hook（0:00–0:20）**

> 面试前、睡前、想更自信时，我们听的都是别人的歌。  
> 如果这一次，安慰你的，是**你自己的声音**呢？

**进入产品（0:20–0:40）**

- 打开 H5（**HTTPS** 或 localhost，否则无麦克风）。
- 首页点 **「新建」**。
- 输入框写 Demo 句，例如：  
  「我明天面试，很怕自己答不好，看到别人都有 offer，觉得自己很差。」
- 点 **「生成肯定语」** → 展示可编辑肯定语（可口播：「不是 chatbot 甩一段鸡汤，是可以改成自己的话」）。

## 2. 核心 WOW：听到自己的声音（0:40–1:40）

- **「用我的声音录下它 →」**
- 朗读屏幕上的肯定语；展示 **单条丝带** 录音反馈。
- 勾选 **本人授权**。
- 选 **自信 Confidence** 背景（可试听）→ 混音台保持默认或略调 → **合成**。
- 四阶段进度走完 → **播放**。  
  > 停顿 5 秒，让评委听到：背景 + 自己的声音（潜听）像一件完整 sub。

## 3. 从玩具到产品（1:40–2:10）

- 结果页：**自动出现在「我的作品」**（可返回首页 Tab 展示封面网格）。
- 演示 **改名、下载、分享卡片**（说明：只分享封面，不含原始倾诉与录音文件）。
- 若评委问「效果怎么衡量」：说明情绪评分在 roadmap，当前强调 **可保存、可复听、可分享** 的闭环。

## 4. 为什么属于腾讯音乐（2:10–2:35）

> 从「平台推荐适合你的歌」到「AI 帮你生成属于你的 sub」——  
> 可落在情绪歌单、K 歌声音资产、睡眠/疗愈场景的 **个性化音频** 位。

## 5. 收尾（2:35–3:00）

- 路径：今天面试 sub → 明天睡前 sub → 作品库 → 分享卡片。
- 金句（看着评委）：  
  > 帮用户把一句「我不行」，变成用自己声音说出的「我可以」。

## 不翻车清单

- [ ] 备份 Demo 视频（手机横屏录屏）
- [ ] 演示机提前授权麦克风 + 外放音量
- [ ] **HTTPS** 隧道（内网 IP 无麦克风）
- [ ] DeepSeek 断网时说明 **本地模板兜底**
- [ ] 录音 ≥3s，建议 10–30s
- [ ] 单标签页，避免抢音频设备
