# 心声调频 InnerTune — 路演 / Demo 脚本（3 分钟）

> 依据「YC / Anthropic / OpenAI 黑客松获奖规律」整理：评委奖励的是**有用 + 意外 + 现场不翻车的 Demo + 清晰故事**，
> 而不是最复杂的工程。把至少 20–25% 的精力放在 Demo 与叙事；功能在还剩 25% 时间时冻结。

## 0. 一句话定位（开场前先让评委记住）
> 我们不是让 AI 替你写一首歌，而是帮你把一句「我不行」，变成一段**用你自己声音说出**的「我可以」。

## 1. WOW 必须在前 30 秒（先放 Demo，不要先放 PPT）

**开场 Hook（0:00–0:20，口播）**
> 面试前、考试前、睡前，我们都靠音乐让自己平静。但那些歌始终是别人写的。
> 如果这一次，安慰你的人，是**你自己**呢？

**立刻进入产品（0:20–0:35）**
- 打开 https://（你的隧道/部署地址）。
- 直接点首页的 **「▶ 一键看一个完整演示」**——它会自动填入「面试焦虑」case 并生成，省去现场打字。
- 评委马上看到「**AI 理解到你现在的状态**」卡片 + 生成策略（降低比较感 / 恢复掌控感 / 强化行动感）。
  > 这一屏就说明：它不是套壳 chatbot，而是「情绪理解 → 文案策略 → 声景生成」的系统。

## 2. 核心 WOW：听到自己的声音（0:35–1:30）

- 点「用我的声音录下它」，把**核心心声**那一句读出来：
  > 「我可以紧张，但我依然可以稳定地表达自己。」
- 勾选「本人授权」→ 选 **自信 Confidence** 配方（可点「▶ 试听」让评委先听到音景）。
- 点「生成我的心声调频」——四阶段生成动画走完（理解 → 对话 → 混声 → 封面）。
- **关键时刻：播放。** 让评委清楚听到：背景音乐 + **你自己的声音** + 自然的进入感，像一个完整作品，不是拼接素材。
  > 这是全场最有情绪冲击力的 10 秒，停顿一下，让它发生。

## 3. 让它从「玩具」变「产品」：情绪前后评分（1:30–2:10）

- 播放前先给「此刻的你」打分（紧张 / 自信 1–5），听完再打一次。
- 展示变化：**紧张 4 → 2，自信 2 → 4**。
  > 这一步是我们的产品闭环：它不只是好玩，而是**可量化用户情绪变化**的陪伴工具。
- 顺手展示封面 +「分享声景卡片」（只分享卡片、不含原始倾诉与声音，隐私安全）+「我的音轨」回顾。

## 4. 为什么属于腾讯音乐（2:10–2:35，主题/赛道对齐——近乎必答）
> TME 的生态不只是听歌，还有听、唱、看、表演和围绕音乐的社交。
> 心声调频是 TME 场景音乐的下一步：**从「平台推荐适合你的音乐」，到「AI 为你生成属于你的音乐」**。
- 落点：QQ 音乐情绪歌单新增「生成我的情绪声景」；全民 K 歌的声音资产变成自我陪伴音频；
  可接入启明星音乐疗愈 / 空间音频 / 语音合成能力。

## 5. 能不能活过黑客松 + 收尾（2:35–3:00）
- **路径**：今天面试声景 → 明天睡前声景 → 一周后「情绪声景日记」→ 私人心声歌单。
- **首批用户 / 验证**（用真实数据，不要编）：我们找了 N 位面试 / 睡前焦虑用户试用，多数表示
  「用自己的声音听到这句话」比普通 AI 旁白更有代入感；据此我们加了本地录音、可编辑肯定语、非公开分享。
- **收尾金句（看着评委说）**：
  > 我们不是让 AI 替用户写一首歌，而是帮他把一句「我不行」，变成用自己声音说出的「我可以」。

## 不翻车清单（现场可靠性 = 第一可控变量）
- [ ] **录一段备份 Demo 视频**（手机横屏即可），现场任何环节卡住立刻切视频。
- [ ] 提前在**演示用的那台设备 + 那个浏览器**跑通一遍：麦克风权限、声音外放、网络。
- [ ] 用 **HTTPS** 访问（隧道/部署），否则浏览器禁用麦克风——这是最容易翻车的点。
- [ ] DeepSeek key 已配置；万一断网，产品会自动用**本地模板**兜底，Demo 仍可继续（可顺势说这是我们的鲁棒性设计）。
- [ ] 麦克风若在评委设备上被拒：点「允许麦克风后 · 重新尝试」，或直接切备份视频。
- [ ] 现场音量调好；录音建议读满 ≥10 秒，太短系统会提示。
- [ ] 全程别开第二个标签页抢音频设备。

## 三件评委一定打分的事（对照检查）
1. **有用 + 意外**：自我对话声景 ≠ 又一个 chatbot；WOW = 听到自己的声音在音乐里说相关的话。
2. **主题对齐**：明确讲「为什么是腾讯音乐 / 音乐赛道」。
3. **清晰叙事 + 团队**：开场痛点 → WOW Demo → 闭环数据 → 生态 → 收尾；点名每个成员的贡献。

## 时间预算（黑客松倒排）
- 还剩 25% 时间：**功能冻结**，只修 Demo 路径上的 bug。
- 最后 20–25% 时间：反复演练 Demo + 录备份视频 + 打磨 30 秒开场。
