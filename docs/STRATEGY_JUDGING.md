# 酥饼 — 黑客松评审对标策略（精简）

> 赛事：Build For Music+ 腾讯音乐 AI Hackathon 2026。产品实现以 **酥饼** 品牌与移动端 H5 为准。

## 一句话结论

不要表达成「AI 生成音乐 + 玄学显化」，而是：

**用 AI 把用户想法转译为自我肯定语，结合本人声音与程序化 sub 背景，生成可保存、可分享的专属 sub 音频。**

参赛打法 = 可运行、可听见、可保存的 **sub 创作闭环**（移动端 demo）。

## 获奖产品共性 → 本项目对策

1. **能跑的东西获奖**：首页 → 新建 → 六步向导 → 作品入库，全链路浏览器内完成。
2. **清晰单场景**：面试 / 考试前 / 睡前的 1–3 分钟 sub（Demo 用自然语言输入即可）。
3. **WOW Moment**：听到**自己的声音**在 sub 里朗读与自己相关的肯定语（潜听混音）。
4. **技术可见**：LLM 肯定语 + Web Audio 混音 + Canvas 封面 + 本地作品库。
5. **延展路径**：今天面试 sub → 明天睡前 sub → 作品列表 → 分享卡片。

## 风险与对策

| 风险 | 对策（已实现） |
| --- | --- |
| 玄学 | Prompt + 路由禁词；外部文案「自我肯定 / 陪伴 / 放松」 |
| 医疗暗示 | 全程非医疗声明 |
| 声音克隆 | 仅本人现场录音 + 授权勾选 |
| 版权 | 程序化 Sound Recipe，零采样 |
| 功能过多 | 单段输入、无社区、无克隆 |

## 最终产品定义（参赛文档可用）

> 酥饼是一款 AI 个性化自我肯定 sub 工具。用户写下此刻的想法，系统生成可编辑的第一人称肯定语；
> 用户录制自己的声音，选择背景音与混音参数，在浏览器内合成一段 sub 音频与封面，并自动保存到「我的作品」，
> 支持下载与隐私安全的卡片分享。

收尾金句：

> **我们不是让 AI 替用户写一首歌，而是帮用户把一句「我不行」，变成用自己声音说出的「我可以」。**

## 三个关键词

Personal（个人化）· Embodied（具身化）· Musical（sub 化，非整曲）。
