# 心声调频 InnerTune — 黑客松评审对标策略（精简）

> 依据主流 Hackathon 评审标准（MLH / TAIKAI / NASA Space Apps / ETHGlobal）、音乐科技 Hackathon 规则、
> 腾讯云 Hackathon 获奖案例与 AI 音乐获奖项目反推。赛事：Build For Music+ 腾讯音乐 AI Hackathon 2026，初稿截止 2026-06-25。

## 一句话结论
不要表达成「AI 生成音乐 + 玄学显化」，而是：
**用 AI 把用户烦恼转译为积极自我对话，结合本人声音、AI 音乐与声景参数，生成专属情绪陪伴音频。**
参赛打法 = 做成可运行、可听见、**可量化**、可分享的「情绪转化闭环」。

## 获奖产品共性 → 本项目对策
1. **能跑的东西获奖，不是好 idea**：交付完整 demo 闭环（已实现 7 步闭环）。
2. **清晰单场景**：主打面试/考试前的 90 秒自信声景；不面向所有人。
3. **WOW Moment**：用户听到**自己的声音**在音乐里说出一句真正相关的话（核心心声 anchor）。
4. **技术包进产品价值**：LLM=把混乱变清晰自我对话；录音=从「别人安慰我」到「我安慰我自己」；情绪前后评分=反馈闭环。
5. **可持续产品化路径**：今天面试声景 → 明天睡前声景 → 情绪声景日记 → 私人心声歌单。

## 风险与已落地的对策
| 风险 | 对策（已实现） |
| --- | --- |
| 玄学 | 系统提示词显式禁止玄学/显化/招财/绝对化；外部文案统一「自我对话/情绪陪伴/放松」 |
| 医疗暗示 | 全程非医疗声明；不提治愈焦虑抑郁失眠 |
| 声音克隆 | 只做本人现场录音 + 授权勾选 + 可删除，不做克隆 |
| 版权 | 声景程序化合成，零采样、零版权 |
| 功能过多 | 收窄为单一闭环，不做社区/克隆/实时音乐生成 |

## 已落地的关键优化（对应建议 6.1–6.7、7、11）
- 6.1 三段输入：现在状态 / 想抵达 / **不想听到**（降低鸡汤感）。
- 6.2 肯定语页「AI 理解 + 生成策略」卡片（展示系统设计深度）。
- 6.3 录音以**核心心声**为主 + 本人授权勾选。
- 6.4 「声景配方 Sound Recipe」框架（Confidence/Calm/Focus/Reset/Sleep + 音乐设计说明）。
- 6.5 四阶段生成可视化（理解→对话→混声→封面）。
- 6.6 **情绪前后评分**（紧张/自信 1–5，展示变化）——产品闭环核心。
- 6.7 隐私安全分享卡片（只含封面/标题/标签/核心心声，不含原始倾诉）。
- 7   LLM 输出结构化 JSON。
- 11  反玄学/反绝对化/反医疗的内容安全约束 + 危机词触发求助。

## 最终产品定义（可放进参赛文档）
> 心声调频 InnerTune 是一款 AI 个性化自我对话声景工具。用户输入当下的烦恼或目标，系统识别情绪状态，
> 生成温和、积极、可编辑的第一人称自我肯定语与一句核心心声。用户录制自己的声音，选择自信/放松/专注等声景配方，
> 系统将人声、背景音乐、环境音与音频参数混合，生成一段 45–90 秒的专属情绪陪伴音频，并记录聆听前后的情绪变化。

收尾金句：
> **我们不是让 AI 替用户写一首歌，而是让 AI 帮用户把一句「我不行」，变成一段用自己声音说出的「我可以」。**

## 三个关键词
Personal（个人化）· Embodied（具身化，用户听到自己的声音）· Musical（音乐化）。
