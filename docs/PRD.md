# PRD ·《酥饼：用你的声音，生成专属于你的显化 sub》

**副标题**：把当下的烦恼，变成一段只属于自己的 sub 音频。  
**版本**：v0.3（移动端 + 玻璃 UI）｜ 状态：核心闭环已跑通

---

## 1. 一句话

用户写下此刻的想法 → AI 生成可编辑肯定语 → 本人录音 → 选背景音 / 调参混音 → 生成 sub 音频 + 封面 → 自动保存到「我的作品」→ 下载 / 分享卡片。

## 2. 目标用户与场景

面试 / 考试前紧张、睡前内耗、自我怀疑、想更自信的年轻人。  
Demo 友好场景：**面试 / 考试前的 1–3 分钟 sub**（情绪强、可听见、可复现）。

## 3. 痛点 → 价值

- 情绪混乱时组织不出积极语言 → AI 生成贴合语境的肯定语（可编辑、可换语气）。
- 文字容易被忽略 → **用自己的声音**录下肯定语，具身化陪伴。
- 普通 AI 音乐同质化 → 生成的是「和自己对话的 sub 仪式」，不是完整流行歌。

## 4. 核心闭环（P0 · 当前实现）

```
首页（我的作品）→ 新建
  → 输入（单段文本，≤200 字）
  → 肯定语（编辑 + 重新优化 / 更柔和 / 更有力量）
  → 录音（Strands 单条丝带可视化 + 本人授权勾选）
  → 背景音（AI 配方 / 上传 / QQ 音乐占位 / 不添加）
  → 混音台（潜听模式：人声低音量循环铺底）
  → 结果（播放 / 改名 / 下载 / 分享卡片 / 再做一条）
  → 自动写入「我的作品」
```

**与 v0.2 文档差异（以代码为准）**：

- 输入页已简化为 **单文本框**「在想什么呢？」（不再展示场景选择、三段输入、示例画廊）。
- 结果页 **自动保存**，当前 **未展示** 聆听前后情绪评分 UI（`EmotionRating` 组件保留但未接入）。
- 首页为 **工作台**（作品 Tab + 社群占位），非直接进入向导。
- 全站 **移动端画框**（480px）+ **浅色玻璃 UI**。

## 5. 核心创新

| 创新 | 说明 |
| --- | --- |
| 自我声音 | 用户听到的是自己读出的肯定语（潜听混音） |
| 情绪转译 | LLM 结构化肯定语 + 本地模板兜底 |
| 程序化 sub | Web Audio 合成背景，零版权采样 |
| 可留存 | 封面 + 音频本地历史（localStorage） |
| 可传播 | 分享卡片（封面 / 标题 / 标签），不含原始倾诉与原始录音 |

## 6. 成功标准

- 手机 WebView 内完成一条 case：输入 → 生成 → 录音 → 混音 → 播放 → 首页可见保存项。
- DeepSeek 不可用时本地模板仍可 demo。
- 合规：反玄学/反医疗、AI 标识、危机词安全、录音授权勾选。

## 7. 技术架构

Next.js 15 App Router · TypeScript · Tailwind v4 · Framer Motion · Web Audio 离线混音 · Canvas 封面 · DeepSeek API（`/api/generate`）+ fallback。

## 8. 非目标（本期不做）

完整声音克隆、整首歌生成、社区 Feed、心理测评、玄学打卡、桌面宽屏独立布局。

## 9. Roadmap

- **P1**：情绪评分 UI 重新接入、TTS 兜底、波形可视化、分享体验优化。
- **P2**：sub 日记、私人心声列表、与 TME 场景（歌单 / K 歌 / 睡眠）联动。
