# 心频 InnerTune — 项目方案（原始策略文档）

> 本文件为产品/策略原始稿，作为 PRD、技术实现与黑客松提交材料的依据来源。
> 赛事：Build For Music+ 腾讯音乐 AI Hackathon 2026 ｜ 初稿截止：2026-06-25

---

## 1. 项目重新定位

不直接叫「AI 显化 sub 工具」，而是改成：

**产品暂名：心频 InnerTune / 许愿电台 / Manifest FM / MeSound**

**一句话介绍：**

把用户当下的烦恼、目标或情绪状态，转化成一段由 AI 生成的专属自我肯定音轨：包含肯定语、本人声线、背景音乐、声音参数和封面图，让用户每天用自己的声音听见想成为的自己。

**黑客松版定位：**

一个结合 **AI 文本生成 + AI/本人声音 + AI 音乐/音频合成 + 情绪陪伴** 的个性化音乐创作工具。

---

## 2. 核心产品价值

**原 idea：** AI 生成音乐 + 玄学显化 sub

**优化后：** AI 生成"自我肯定音频 ritual"

它解决的不是"帮用户实现愿望"，而是：把混乱情绪转成可反复收听的自我肯定音频。

差异点：**AI 个性化生成 + 本人声音 + 音乐参数化 + 可保存成作品**。

---

## 3. 外部表达要避开的词和替换词

| 不建议主打 | 建议替换 |
| --- | --- |
| 显化、招财、转运、复合、疗愈疾病 | 自我肯定、目标提醒、情绪陪伴、放松练习 |
| 潜意识植入、不可察觉暗示 | 用户可见、可听、可编辑的肯定语音轨 |
| 高频能量、神秘频率、保证改变现实 | 声音氛围、节奏参数、沉浸式音景 |
| 治愈焦虑/抑郁/失眠 | 帮助放松、睡前陪伴、非医疗心理支持 |
| 克隆任意人声音 | 仅使用本人声音或明确授权声音 |

最好的包装句：**"不是许愿神器，而是把目标、情绪和自我肯定变成一段每天可以听见的私人音乐练习。"**

## 4. MVP 闭环设计

### MVP 必做版本

1. **用户输入状态/烦恼** — 例如："我最近找工作很焦虑，总觉得自己不够好。"
2. **AI 生成肯定语脚本** — 标题、情绪标签、6-8 句肯定语、适合的声音氛围。
3. **用户录制自己的声音** — MVP 先直接录制，不做完整 voice clone。
4. **选择/上传背景音乐** — 3-5 个版权安全内置音景：睡前、专注、勇气、放松、晨间。
5. **音频合成** — 人声 + 背景音乐 + 轻节奏/空间感参数，输出 30-90 秒音频。
6. **AI 生成封面图 + 保存播放** — 生成"我的今日音轨"封面。

### MVP 不建议做的功能
- 完整 AI 声音克隆（合规/调试/稳定性复杂）
- 直接接 Suno API 自动生成（授权链路风险）
- 用户上传任意流行歌做 BGM（版权风险）
- 社区 Feed（审核压力）
- 长音频/整首歌生成（demo 只需 30-90 秒）

## 5. 更完整的用户流程

### 首屏
标题：**今天想把什么变成你的声音？**
输入提示：写下一个烦恼、一个目标，或一句你想对自己说但说不出口的话。
按钮：**生成我的肯定语音轨**

### AI 生成页
输出结构：音轨标题 / 适合场景 / 情绪标签 / 6 句肯定语。
操作：重新生成 / 改温柔一点 / 改坚定一点 / 录制声音。

### 录音页
提示：请用自己的声音读出这些句子。这段声音只用于生成你的私人音轨，不会默认公开。
按钮：开始录音 / 重录 / 下一步。

### 音景选择页
| 音景 | 适合场景 |
| --- | --- |
| 睡前雾蓝 | 睡前放松 |
| 晨间微光 | 起床鼓励 |
| 低谷拥抱 | 情绪支持 |
| 面试勇气 | 行动前打气 |
| 专注白噪 | 学习工作 |

参数包装：声音距离（近/中/远）、人声强度（清晰/轻声/环绕）、节奏感（无/轻微/明显）、氛围（温柔/空灵/坚定/明亮）。
双耳节拍、频率仅作"声音设计参数"，不宣称确定疗效。

## 6. 技术实现方案

| 模块 | MVP 做法 | 技术难度 |
| --- | --- | --- |
| 前端 | Web/H5，小程序原型也可 | 中低 |
| 文本生成 | 调用任意 LLM API，生成肯定语、标题、音景 prompt | 低 |
| 录音 | 浏览器 MediaRecorder / 移动端录音组件 | 中低 |
| 音频处理 | Python + FFmpeg / pydub 混音、淡入淡出、音量归一化 | 中 |
| 背景音乐 | 内置 3-5 段版权安全 loop | 低 |
| 封面图 | 图像生成 API 或预设模板 + 文案 | 低 |
| 播放保存 | 生成 mp3/m4a，页面播放 | 中低 |

推荐架构：用户输入 → LLM 文案生成 → 肯定语编辑页 → 浏览器录音 / TTS fallback → 音频合成 → 封面生成 → 音轨播放页。

核心一句话：**"MVP 只需要做一个 Web Demo：输入文本 → 生成肯定语 → 录音 → 自动混背景音乐 → 生成 60 秒音频 → 播放保存。核心是音频合成和 Demo 闭环，不需要训练模型。"**

## 7. 合规与安全设计

| 风险 | 产品处理方式 |
| --- | --- |
| 玄学/迷信 | 外部文案统一为"自我肯定、情绪陪伴、放松练习"，不承诺结果 |
| 心理健康 | 明确"非医疗/非心理诊疗"，遇自伤/他伤/严重抑郁输入触发安全提示 |
| 声音克隆 | 只允许本人声音或明确授权声音，禁止上传他人声音 |
| AI 内容 | 音频、封面、文案均标注"AI 辅助生成" |
| 版权 | Demo 使用内置版权安全音乐；用户上传需声明拥有使用权 |
| 隐私 | 默认不公开烦恼文本和声音素材；提供删除入口 |

合规依据：《互联网信息服务深度合成管理规定》《生成式人工智能服务管理暂行办法》《人工智能生成合成内容标识办法》（2025-09-01 施行，音频需在起始/末尾/中间加语音或节奏提示，或界面显著提示）。

产品内固定提示：
> 本音频由 AI 辅助生成，仅用于个人放松、自我肯定和情绪记录，不构成医疗或心理诊疗建议。
> 本功能仅支持使用本人声音或已获明确授权的声音素材。

## 8. 黑客松评审叙事

- **问题**：年轻人不缺歌听，缺的是把"我现在很乱"转化成"我可以被自己支持"的低门槛方式。
- **方案**：AI 音频生成工具 — 输入烦恼 → AI 转肯定语 → 本人声音录制 → 加音乐/氛围/封面 → 专属心理暗示音轨。
- **为什么是音乐赛道**：音乐承载情绪，声音比文字更有陪伴感；从"用户听音乐"拓展到"用户生成自己的情绪音乐"。

---

## 9. Demo 场景设计（3 个固定 case）

- **Case 1 面试焦虑**：输入"我明天有面试，很怕表现不好。" → 《我可以稳定表达自己》/ 晨间微光 / 坚定 / 中速。
- **Case 2 睡前内耗**：输入"我晚上总是想很多，觉得自己不够好。" → 《今晚我允许自己休息》/ 睡前雾蓝 / 轻声 / 慢节奏。
- **Case 3 目标显化安全包装**：输入"我想变得更自信，更敢争取机会。" → 《我正在靠近更自信的自己》/ 明亮 / 温柔坚定 / 轻节奏。

## 10. 功能优先级

### P0（必须完成）
用户输入烦恼/状态 · AI 生成肯定语 · 用户录音 · 背景音乐选择 · 自动混音 · 生成封面 · 播放生成音频 · 基础合规提示。

情绪日记 · 每日播放打卡 · 个性化歌单 · 匿名社区 · 朋友互赠肯定语 · 创作者/心理咨询师/音乐人模板 · 与 K 歌/歌单/播客/睡眠音频场景结合。

## 15. 最强差异点

**AI 生成的不是一首歌，而是一个"用户和自己对话的声音仪式"。**

初稿阶段把 **"输入烦恼 → 生成肯定语 → 录自己的声音 → 混成专属音频 → 播放保存"** 闭环跑通即可。
