# 酥饼 — 项目方案（策略与产品限定）

> 本文件为产品/策略依据，需与当前代码和 [`DESIGN_SYSTEM.md`](DESIGN_SYSTEM.md) 保持一致。  
> 赛事背景：Build For Music+ 腾讯音乐 AI Hackathon 2026

---

## 1. 产品定位

**产品名：酥饼**

**一句话：**

用你的声音，生成专属于你的显化 sub——把当下的烦恼或期许，变成一段可反复收听的自我肯定音频。

**对外表达：**

- 主打：**自我肯定 sub / 情绪陪伴 / 放松练习**
- 避免主打：玄学显化、招财转运、医疗疗愈、潜意识植入

黑客松叙事仍可用：「AI 文本 + 本人声音 + 程序化音频 + 封面作品」的个性化创作工具。

---

## 2. 核心产品价值

| 原 idea | 当前限定 |
| --- | --- |
| AI 音乐 + 玄学 sub | **AI 自我肯定 sub**：可见、可听、可编辑 |
| 帮用户「实现愿望」 | 帮用户把混乱情绪变成**可朗读、可混音、可保存**的 sub |
| 桌面 Web 演示 | **移动端 WebView / H5** 优先（480px 画框） |

差异点：**个性化肯定语 + 本人录音 + 零版权程序化背景 + 玻璃风作品页**。

---

## 3. 用词替换（合规）

| 不建议主打 | 建议替换 |
| --- | --- |
| 显化、招财、转运、复合、疗愈疾病 | 自我肯定、情绪陪伴、放松练习、sub 音频 |
| 潜意识植入 | 用户可见可编辑的肯定语 |
| 神秘频率、保证改变现实 | 声音氛围、混音参数（不宣称疗效） |
| 治愈焦虑/抑郁/失眠 | 日常陪伴、非医疗声明 |
| 克隆任意人声 | 仅本人现场录音 + 授权勾选 |

产品内可保留「显化 sub」作为用户圈层用语，但界面与生成策略须遵守 [`AFFIRMATION_GUIDELINES.md`](AFFIRMATION_GUIDELINES.md) 的反玄学/反医疗约束。

## 4. MVP 闭环（当前版本）

1. **首页** — 我的作品（2 列封面）/ 社群占位；「新建」进入向导。
2. **输入** — 单段文本（≤200 字），「在想什么呢？」。
3. **肯定语** — AI 生成，可编辑；重新优化 / 更柔和 / 更有力量。
4. **录音** — 朗读肯定语；Strands **单条**丝带反馈；本人授权勾选。
5. **背景音** — AI 配方（Sound Recipe）/ 上传 / QQ 音乐占位 / 无。
6. **混音台** — 潜听模式参数；离线合成 MP3 + 封面。
7. **结果** — 播放、改名、下载、分享卡片；**自动保存**到本地历史。

### 仍不做

- 完整 voice clone、Suno 整曲、用户上传流行歌作 BGM、社区 Feed、长音频（>30min 非目标场景）。

## 5. 用户流程（与 UI 对齐）

| 步骤 | 当前界面 copy |
| --- | --- |
| 首页 | 酥饼 + Tagline；Tab「我的作品 / 社群」 |
| 输入 | 「在想什么呢？」→「生成肯定语」 |
| 肯定语 | 「这是为你生成的肯定语」→「用我的声音录下它 →」 |
| 录音 | 「用自己的声音，坚定地读出你的肯定语」；丝带可视化 |
| 背景音 / 调参 | 选来源 → 混音台 → 合成 |
| 结果 | 「你的所愿变成了一首只属于你的音乐」；再做一条 |

**已下线 / 未接入的 v0.2 设计**：场景 chips、三段输入、示例画廊、一键演示、步骤 EmotionRating 结果页。

## 6. 技术方案（已实现方向）

| 模块 | 做法 |
| --- | --- |
| 前端 | Next.js 15 H5，`.app-frame` 移动端 |
| 文案 | DeepSeek 结构化 JSON + `fallback` 本地模板 |
| 录音 | MediaRecorder + Analyser（level → Strands） |
| 音频 | 浏览器 Web Audio 离线混音 + lamejs MP3 |
| 背景 | 程序化 Sound Recipe，零采样 |
| 封面 | Canvas 程序化 PNG |
| 存储 | localStorage 历史 + 封面缩略图 |

## 10. 最强差异点（不变）

**AI 生成的不是一首歌，而是一段「用户和自己对话」的 sub 仪式。**

闭环：**输入 → 肯定语 → 录自己的声音 → 混成 sub → 保存播放。**
